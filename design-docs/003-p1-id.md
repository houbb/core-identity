# P1 的交付标准（Definition of Done）

P1 完成后，不要求用户能够登录，而要求**整个 Core Platform 已经拥有统一的身份模型**。

具体来说，应满足以下条件：

* **工程层面**

  * `core-identity` 四个子项目（backend、frontend、admin-backend、admin-frontend）均已建立并可运行。
  * 数据库自动初始化（SQLite）。

* **领域层面**

  * 完成 `Identity`、`Account`、`Credential`、`Profile` 四个核心领域对象及其关系。
  * 完成对应 Repository、Service、REST API。

* **用户端**

  * 提供个人资料（Profile）查看与编辑页面。
  * 提供账号（Account）列表与新增/删除入口（暂不涉及登录验证）。

* **管理端**

  * 提供 Identity、Account、Credential 的列表、详情、状态管理页面。
  * 统一使用表格 + 详情页 + Tab 的交互模式。

这样，P2（Authentication Runtime）只需要在现有的 `Account + Credential` 基础上增加认证流程（邮箱验证码、密码、JWT、Refresh Token 等），而无需修改 P1 的领域模型。这也是按 Runtime 分阶段设计最大的价值：**每一阶段都在上一阶段的基础上自然演进，而不是推倒重来。**
